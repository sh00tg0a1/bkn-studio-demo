# 1. 文档概述

## 1.1 编写目的

本文档旨在定义「PMC 智能问答系统」的产品需求，明确系统功能边界、核心使用场景及技术接口要求，作为产品设计、研发实现与测试验收的基准依据。

该系统面向制造业 PMC（生产计划与物料控制）团队，通过自然语言交互方式，帮助 PMC 人员快速获取产品库存、物料需求、交期分析及生产计划等多维度信息，提升业务响应效率，同时赋能业务领导在周会等场景下实时获取供应链数字洞察。

## 1.2 适用范围

- 主要用户群体：PMC 计划员、物料管理员、业务领导（周会决策支持）
- 系统边界：覆盖需求预测、MRP 运算、物料 PO、库存管理、生产日报等核心业务场景
- 不包含：ERP 系统后台管理、财务结算、HR 系统

## 1.3 背景与痛点

当前 PMC 团队在支持业务领导周会时存在以下典型问题：

- 信息分散：库存、PO 交期、缺料情况分布在多个系统（ERP/WMS/供应商门户）中，人工汇总耗时长
- 响应滞后：业务领导提问时，PMC 需临时手动查询并整理，无法实时响应，决策效率低
- 分析浅层：现有报表以固定模板为主，缺乏面向复杂交期分析、备货周期计算等场景的动态分析能力
- 报告生成慢：日报、汇报材料需大量人工填写，格式不统一，质量参差不齐

# 2. 产品定位与目标

「PMC 智能问答系统」是面向制造业供应链场景的 AI 驱动智能助手，以自然语言为交互界面，以 ERP/WMS/SCM 数据为基础，提供从基础信息查询到复杂分析报告的端到端问答能力。

## 3.2 典型使用场景描述

### 场景一：业务领导周会实时问答

周一上午 9:00，业务领导主持销售与运营协调会议（S&OP 周会）。会议中临时提问：「T01-000080 这批产品最晚什么时候能交货？」PMC 计划员通过系统输入问题，30 秒内获得结构化回答（预计交期、关键卡点物料、风险说明），并在会议中直接投屏展示。

### 场景二：PMC 每日生产计划日报生成

每日 8:00，PMC 计划员通过系统输入「为产品 T01-000080 生成今日生产计划日报」，系统自动调取 BOM、物料库存、PO 交期、缺料数据，生成结构化日报（含汇总、风险物料、行动项、14 天到货预测），PMC 复核后一键推送至飞书工作群。

### 场景三：可生产数量综合分析报告

供应链管理者需了解某产品当前最多可生产多少台，以支持客户报价。系统基于产品 BOM、各层物料库存（含替代料）、供应商在途 PO，自动计算最大可生产数量与最小补货缺口，以商务网页报告形式输出。

## 4.2 核心功能详细说明

### F1：基础信息查询

- 产品库存查询：按 SKU/产品编码查询实时库存（含在库、在途、冻结）
- 物料库存查询：查询原材料/半成品库存，支持多仓库汇总
- 需求预测单查询：查看产品的需求预测单号、数量、日期
- 物料 PO 查询：查询 PO 单号、数量、供应商、承诺交期、实际到货状态
- 供应商信息查询：供应商名称、联系方式、负责物料范围、历史准时率

### F2：MRP 状态分析

- MRP 投放状态：判断某产品需求预测是否已触发 MRP 运算，运算时间与版本
- 缺料汇总分析：展示缺料物料数量、缺料量、预计缺料影响产品数量
- PO 交期分析：计算各 PO 的预计到货时间与需求时间差，标记逾期风险

### F3：交期复杂分析

- 批次预计交货时间：综合考虑缺料 PO 到期、生产提前期，计算产品最早可完工日期
- 交付卡点识别：识别关键路径上的卡点物料（影响交期的长周期/缺料物料）
- 风险物料预警：识别存在供货风险的物料，输出风险级别与建议行动
- 备货周期计算：基于历史到货周期、生产提前期计算产品备货所需总时长
- 本月生产/交付问题产品扫描：批量扫描本月所有产品，输出有交付风险的产品列表

### F4：研究与报告生成

- 可生产数量综合分析报告：基于 BOM 多层展开，结合各物料库存（含替代料）与在途 PO，计算最大可生产量、最小补货量，以商务网页格式输出
- 生产计划日报（飞书格式）：
	- 日报总体情况：产品名称、需求预测单号、BOM 数量及层级、自制件 MRP 状态、齐套率、风险物料清单
	- 高优先级行动项：待完成 PR/PO 的物料清单（含对象、原因、建议）
	- 未来 14 天到货总量预测与风险提示
	- 所有结论必须附数据来源标注（系统来源 + 单据编号）

## 6.2 数据质量要求

- 实时性：库存台账数据延迟 ≤ 1 小时；PO 状态数据延迟 ≤ 4 小时；MRP 结果数据以最新一次运算为准
- 完整性：BOM 数据必须包含完整的多层结构及替代料配置；物料主数据必须包含采购周期字段
- 一致性：产品编码、物料编码在各数据源（ERP、WMS、MRP 系统）中保持唯一，使用统一编码体系
- 可溯源：所有报告输出的数据结论必须标注数据来源（系统名称、单据编号、截取时间）

# 7. 非功能需求

## 7.1 性能要求

- 基础查询类问题（F1/F2）：系统响应时间 ≤ 10 秒
- 复杂分析类问题（F3）：系统响应时间 ≤ 30 秒
- 报告生成类（F4）：生产计划日报 ≤ 60 秒；网页分析报告 ≤ 120 秒
- 系统并发：支持 ≥ 20 用户同时使用，响应时间不得超过上述标准的 2 倍

## 7.2 准确性要求

- 数值类问题（库存、PO 数量）：准确率 100%（实时对齐源数据）
- 分析类问题（交期预测、缺料识别）：准确率 ≥ 90%（与人工分析结果对比）
- 报告类输出：所有数据字段必须有来源标注，禁止 AI 凭空生成数字

## 7.3 安全与权限

- 角色权限：业务领导只读权限；PMC 计划员读写权限；管理员全权限
- 数据脱敏：对外输出报告中的供应商价格信息需脱敏处理
- 操作日志：所有查询和报告生成操作需记录日志（用户、时间、问题内容、输出摘要）

## 7.4 可用性

- 系统可用性 SLA：≥ 99.5%（非计划停机 ≤ 43.8 小时/年）
- 支持移动端（飞书小程序/H5）访问
- 日报推送功能支持接入飞书 Bot，实现自动推送到指定飞书群

# 8. 界面与交互要求

## 8.1 主交互界面

- 提供类 ChatGPT 对话式交互界面，支持自然语言输入
- 对话历史记录持久化，支持回溯历史问答
- 支持快捷问题模板（预设常用业务问题，一键触发）
- 复杂分析结果支持在对话界面内渲染结构化表格和图表

## 8.2 报告输出格式

- F4 可生产分析报告：以商务风格 HTML 网页形式输出，支持下载为 PDF
- F4 生产计划日报：输出两种格式——①系统内展示的结构化页面；②飞书 Markdown 格式（用于 Bot 推送）
- 所有报告底部须包含「数据生成时间」与「数据来源系统」标注

## 8.3 错误处理

- 当系统查询不到相关数据时，明确告知用户「未找到相关数据，请确认产品编码/物料编码是否正确」，不得凭空返回结果
- 当 MRP 尚未运算时，返回「该需求预测单尚未完成 MRP 运算，请先在 ERP 中触发 MRP」
- 当数据来源存在冲突时，以 ERP 系统数据为最终依据，并在回答中注明「数据存在差异，以 ERP 数据为准」
